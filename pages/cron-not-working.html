Cron Not Working? Troubleshooting Guide

Your cron job isn't executing? Here are the most common reasons and solutions.

🔍 Common Issues

1. Missing Execute Permission

Your script needs executable permissions:

chmod +x /path/to/script.sh

2. Wrong PATH

Cron has a minimal PATH. Use absolute paths:

# Wrong
* * * * * script.sh

# Correct
* * * * * /usr/local/bin/script.sh

3. Missing Shebang

Your script needs a proper shebang:

#!/bin/bash
# Your code here

4. Environment Variables

Cron doesn't load your .bashrc or .profile. Set variables explicitly:

* * * * * PATH=/usr/local/bin:/usr/bin /path/to/script.sh

🧪 How to Debug

Check Cron Logs

# Ubuntu/Debian
tail -f /var/log/syslog | grep cron
# CentOS/RHEL
tail -f /var/log/cron
# macOS
log show --predicate 'process == "cron"' --last 1m

Test Your Command Manually

# Run exactly what cron would run
/bin/bash -c "your-command-here"
# Or run the script directly
/path/to/script.sh

Redirect Output to Log

* * * * * /path/to/script.sh >> /tmp/script.log 2>&1

📋 Cron Service Status

Check if Cron is Running

systemctl status cron # CentOS systemctl status crond sudo launchctl list | grep cron

Restart Cron

sudo systemctl restart cron sudo systemctl restart crond

⚠️ Special Characters Gotchas

Percent Sign (%)

Use backslash to escape % in commands:

* * * * * echo "Date is %Y-%m-%d" >> log.txt # Correct * * * * * echo "Date is \%Y-\%m-\%d" >> log.txt

Question Mark (?)

Some systems don't support ?. Use * instead:

# Use this instead of ?
* * * * *

✅ Quick Checklist

  • ☑️ Cron service is running
  • ☑️ Script has execute permission (chmod +x)
  • ☑️ Using absolute paths
  • ☑️ Script works when run manually
  • ☑️ Output redirected to log file
  • ☑️ Checked cron logs for errors
